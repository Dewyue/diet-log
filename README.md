# 饮食日志

按「一份」快速记录饮食的个人追踪工具：对照每日目标看热量区间，数据存在本机浏览器。

在线：https://dewyue.github.io/diet-log/

## 相对常见饮食记录 App 的优势

- **按「一份」记，不靠称重搜菜**  
  一份约 90 kcal，对应主食 / 蛋白 / 蔬菜等宏量比例，类似「一拳主食、一拳肉、两拳菜」：先判断类型、再记份数，日常更快、更稳。

- **食物交换份思路，加减清晰**  
  各类模版热量对齐，份数可半份递增，一天吃了几份一目了然，方便规律复盘。

- **热量三档一眼看懂**  
  未到目标（蓝）、目标～1800（绿）、超过 1800（红）；1800 为固定上限，目标可自设，状态不靠猜。

- **热量由宏量推导**  
  按蛋白×4 + 碳水×4 + 脂肪×9 计算，改成分总热量自动变，少手填、少算错。

- **今日与历史分工清楚**  
  今日页只看当天进度与餐食；数据页用日历回看，格子同色表示当天热量区间，点进可看明细。

- **记录路径极短**  
  右下角一键添加，选模版 → 调份数 → 保存，适合随手记。

- **数据在本机，可备份**  
  浏览器本地存储，支持导出 / 剪贴板备份；无账号、无云同步依赖。

- **可装成 PWA**  
  手机主屏幕打开，接近轻量 App，随时记一餐。

- **界面克制**  
  只保留目标、份数、进度与日历，没有广告、社交与复杂营养库干扰。

## 功能

- 今日：热量进度、蓝 / 绿 / 红状态、按时间列出餐食
- 按份模版：纯蛋白、瘦肉蛋、主食、蔬菜、奶类、油脂等
- 数据：月汇总 + 日历着色，点日期查看当天记录
- 设置：每日目标、备份导入导出、清空数据

## 本地开发

```bash
bun install
bun run dev
```

## 构建与部署

```bash
bun run build
```

推送到 `main` 后 GitHub Actions 发布到 Pages（`VITE_BASE_PATH=/diet-log/`）。
